# Nibbler Docker deployment

This deployment keeps the live ComfyUI LoRA library on Bender and mounts it read-only into the LoRA Builder containers on Nibbler.

## Host layout

Expected Nibbler root:

```text
/home/clink/docker/lora_builder
├── app/      # git clone lives here
├── data/     # persistent sqlite DB
├── backups/  # copied/imported DB backups
├── secrets/  # SMB credential file, not committed
└── mounts/
    └── loras # CIFS mount of //BENDER/LoRA_Library
```

## Runtime paths

The backend wrapper `Database/backend/lora_api_server_docker.py` maps these environment variables into the existing backend scripts:

```text
LORA_ROOT=/loras
LORA_DB_PATH=/data/lora_master.db
```

The compose stack mounts:

```text
/home/clink/docker/lora_builder/data         -> /data
/home/clink/docker/lora_builder/mounts/loras -> /loras:ro
```

## First run

From Nibbler:

```bash
cd /home/clink/docker/lora_builder/app/LoRA_weights_builder/deploy/nibbler
cp .env.example .env
docker compose --env-file .env up -d --build
```

Then test:

```bash
curl -s http://127.0.0.1:5174/health
```

The direct LAN test URL is:

```text
http://192.168.0.10:5174
```

## DNS / NPM target

Once the stack is healthy:

```text
lora_builder.clink2012.uk -> 192.168.0.10
```

Nginx Proxy Manager should forward to either:

```text
host: 192.168.0.10
port: 5174
scheme: http
```

or, when the proxy host is attached to the same external Docker network:

```text
container: lora_builder_ui
port: 80
scheme: http
```

## Notes

- The LoRA share should remain read-only to protect the live ComfyUI library.
- The database lives under `/home/clink/docker/lora_builder/data` and can be regenerated by reindexing.
- This deployment layer intentionally does not change Phase 8.5 orchestration logic.
